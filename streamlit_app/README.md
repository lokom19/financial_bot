# Trading Signals Dashboard

Интерактивный дашборд для визуализации торговых сигналов на основе ML-моделей.

## Возможности

- **Главная страница**: Сигналы BUY/SELL с консенсусом моделей и системой грейдов
- **Фильтрация**: По минимальному консенсусу и грейду надёжности
- **Сортировка**: По грейду, консенсусу или ожидаемому изменению цены
- **Детальная таблица**: Все сигналы от каждой модели

## Установка

```bash
# Из корневой директории проекта
cd streamlit_app
pip install -r requirements.txt
```

## Запуск

```bash
# Из корневой директории проекта
streamlit run streamlit_app/app.py

# Или через make
make dashboard
```

Дашборд будет доступен по адресу: http://localhost:8501

## Система грейдов

Каждый торговый сигнал получает **грейд от A до F**, который отражает надёжность предсказания. Грейд помогает понять, насколько можно доверять сигналу.

### Что означает каждый грейд

| Грейд | Надёжность | Описание | Рекомендация |
|:-----:|:----------:|:---------|:-------------|
| **A** | Отличная | Высокий консенсус моделей, качественные метрики | Сильный сигнал, можно следовать |
| **B** | Хорошая | Хороший консенсус, метрики выше среднего | Надёжный сигнал |
| **C** | Средняя | Умеренный консенсус, средние метрики | Требует дополнительного анализа |
| **D** | Низкая | Слабый консенсус, метрики ниже среднего | Высокий риск, осторожно |
| **F** | Очень низкая | Минимальный консенсус или плохие метрики | Не рекомендуется следовать |

### Как формируется грейд

Грейд рассчитывается как **взвешенная комбинация** четырёх метрик:

| Метрика | Вес | Что измеряет |
|---------|:---:|:-------------|
| **Консенсус моделей** | 35% | Процент моделей, согласных с сигналом |
| **R² (коэффициент детерминации)** | 25% | Качество модели - насколько хорошо она объясняет данные |
| **Direction Accuracy** | 25% | Точность предсказания направления движения цены |
| **Win Rate** | 15% | Процент прибыльных сделок на историческом бэктесте |

### Пороговые значения

Каждая метрика оценивается по шкале A-F на основе пороговых значений:

| Метрика | A (отлично) | B (хорошо) | C (средне) | D (слабо) | F (плохо) |
|---------|:-----------:|:----------:|:----------:|:---------:|:---------:|
| **Консенсус** | ≥80% | ≥60% | ≥40% | ≥20% | <20% |
| **R²** | ≥0.70 | ≥0.50 | ≥0.30 | ≥0.10 | <0.10 |
| **Direction Accuracy** | ≥65% | ≥55% | ≥50% | ≥45% | <45% |
| **Win Rate** | ≥60% | ≥55% | ≥50% | ≥45% | <45% |

### Формула расчёта

```
Общий грейд = round(
    Грейд_консенсуса × 0.35 +
    Грейд_R² × 0.25 +
    Грейд_direction × 0.25 +
    Грейд_winrate × 0.15
)
```

Где каждый грейд имеет числовое значение: A=5, B=4, C=3, D=2, F=1.

### Пример расчёта

```
SBER - BUY сигнал:
├── Консенсус: 43% (3/7 моделей)
│   └── 43% ≥ 40% → Грейд C (3 балла)
├── R²: 0.45
│   └── 0.45 ≥ 0.30 → Грейд C (3 балла)
├── Direction Accuracy: 58%
│   └── 58% ≥ 55% → Грейд B (4 балла)
├── Win Rate: нет данных
│   └── Грейд F (1 балл), но не учитывается
│
└── Расчёт (без win_rate, веса перенормированы):
    Сумма = 3×0.35 + 3×0.25 + 4×0.25 = 1.05 + 0.75 + 1.0 = 2.8
    Делим на сумму весов: 2.8 / 0.85 = 3.3
    Округляем: round(3.3) = 3 → Грейд C
```

### Рекомендации по использованию

1. **Грейд B и выше** - рекомендуется для принятия торговых решений
2. **Грейд C** - требует дополнительного анализа и подтверждения
3. **Грейд D и F** - высокий риск, лучше воздержаться

### Важно понимать

- Грейд **не гарантирует** прибыльность сделки
- Даже сигнал с грейдом A может оказаться ошибочным
- Используйте грейд как **один из инструментов** принятия решений
- Всегда учитывайте **риск-менеджмент** и диверсификацию

- Обученные модели (запустите `make train` в основном проекте)

```env

DB_PORT=5432
DB_NAME=postgres
```
